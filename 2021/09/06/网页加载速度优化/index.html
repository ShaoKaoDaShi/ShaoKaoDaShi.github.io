网页加载速度优化

发表于2021-09-06|更新于2021-09-06
|阅读量:

网页加载速度优化–图像加载：

  1. 关于 通过 git + hexo 上传文件的问题， 每次改完主题，以本地服务器的显示为主 ， 向github 上传文本文件 github加载速度很快， 但是上传 图片需要等个10分钟左右才能加载出来，查看了一下文件大小，图片大小640*640 jpg文件小大概在 :20-80k， png文件图片大，大概在 300k-800k

  2. 网页加载速度取决于页面中元素内容的总大小， 一个文件html， 文件中加入了 大量的图片， 那么这些图片就会影响网页的加载速度， 所以尽量在保证质量的情况下， 减小图片的文件大小，jpg 格式的图片就比较合适，同时要考虑到 图片在网页上能够显示的最大面积， 从而合理优化图片的大小，让网页速度加载更加快速

  3. 关于html img中src=“ 网络图片地址/本地图片地址” 用户访问的时候有什么区别：

    1. 网页中图片加载速度和什么有关：

      1. 本机带宽的速度
      2. 服务器带宽的速度
      3. 图片大小

    2. 一张图片png 大小为1M，我有一台服务器 带宽为1M ，客户机带宽2M，同时我把图片存在百度图库中，百度图库服务器带宽100M，用两种 src用两种方式来加载， 一种时相对路径（服务器上)， 一种超链接的方式（图片在百度图库服务器中）

      这种情况下服务器 只需要加载 几个字节的问题，图片具体由百度图库服务器来传输

      图片完全由自己的服务器传输， 如果自己的服务器带宽太小的话，整个网页的加载的速度就会变得很慢，文字可以先加载出来，但是图片要响应很长时间

bit，byte，位，字节，汉字的关系：

  1. 字节(Byte)：通常将可表示常用英文字符8位二进制称为一字节。
    • 一个英文字母(不分大小写)占一个字节的空间
    • 一个中文汉字占两个字节的空间
    • 英文标点占一个字节
    • 中文标点占两个字节
  2. 位/比特(bit)：
    • 二进制数系统中，每个0或1就是一个位(bit)，位是数据存储的最小单位。
    • 计算机中的CPU位数指的是CPU一次能处理的最大位数, 例如32位计算机的CPU一次最多能处理32位数据。
  3. 单位的换算为：
    • 1TB = 1024GB
    • 1GB = 1024MB
    • 1MB = 1024KB
    • 1Byte = bit

图片大小计算方法：

  1. 假如手机分辨率是1920×1080，即有这么多个像素点，每个像素点采用RGB（三原色）三个数值记录该点颜色，三个数值大小范围涉及另一个概念色深。
  2. 以24位色深为例，即RGB的每一个值用8位二进制数表示（8×3=24），即每一种颜色的范围为0～2的8次方。所以每一个像素点存储时要占24位，也就是3字节（8位1字节），所以
  3. 图片理论上的存储大小就是图片的像素点数乘以每个像素点的所占存储空间
  4. 不过实际图片用不了这么大存储空间，，因为各种格式的图片还涉及一个关键技术压缩，不同的压缩技术也就导致压缩后图片的存储大小不同。举个不太恰当的例子，用坐标表示每个像素位置，假如像素点（1，2）的颜色值是（12，13，14），像素点（1，3）的颜色值也是（12，13，14），存储的时候就只需要存储一组颜色值，另一个点的颜色值就“同上”，这样就节省了存储空间，压缩就是类似于这样的处理。肉眼观察你所截的图，也能很明显感觉到，画面越丰富，色彩越多，占空间就大，画面越单调，色彩越小，占空间就少。
